# why.fi Requirements

## v1 Requirements

### Core UI Structure
- [ ] **UI-01**: User sees a large live webcam feed as the main focal area
- [ ] **UI-02**: User sees a strong "Target Expression" module clearly showing the required target
- [ ] **UI-03**: User sees a reactive meme panel/board that updates with the currently detected emotion
- [ ] **UI-04**: User can always view live metrics (Why score, Boredom, Confusion, Dread, Detected face, Progress)
- [ ] **UI-05**: User sees a status banner / mission text / system voice messaging
- [ ] **UI-06**: User sees a gallery/strip of captured successful photos

### UX & Interactions
- [ ] **UX-01**: User experiences immediate, rewarding detection feedback
- [ ] **UX-02**: User experiences an intentional, visible cooldown state after matches
- [ ] **UX-03**: User experiences a celebratory and funny match confirmation
- [ ] **UX-04**: User sees meaningful, game-like motion for state changes (targets, detections, matches, scores)

### Screens
- [ ] **SCR-01**: User can navigate and play on the redesigned main gameplay screen
- [ ] **SCR-02**: User views the final judgment / results screen with average scores and captured evidence
